Overview

The goal of the project is to provide Java class wrappers for the IBM ™ z/VM ™ Virtual Machine Operating System's Systems Management Application Programming Interface (which we call for short VSMAPI).

The classes and programming model are intended:

  • to be reliable
  • to be easy to use and program
  • to support both VSMAPI Functions and end-user custom functions
    • with of course support on the host side which allows insertion of custom functions in the VSMAPI server
  • to support the development of higher-level libraries such as packaging PigIron functionality in GUI widgets
  • to support development of GUI applications in terms of such GUI widgets

Note: PigIron is not associated with the IBM corporation, which owns most or all of the trademarks mentioned in PigIron documentation.

NOTE ON SECURITY (LACK THEREOF)

Currently there is NO secure sockets implementation. Sends your password in CLEAR TEXT. Use ONLY on a secure LAN behind a firewall.

Current Level of Support

PigIron currently supports the z/VM R5.3 level of VSMAPI.

Future Development Plans

Additionally to the basic class library, there are visual widgets planned based on PigIron's class embodiments of VSMAPI functions so that VSMAPI functions can be dragged and dropped into some yet-to-be-defined workspace and executed individually or in ordered collections as if they were scripts.

Specification

PigIron is a library of Java class wrappers for the IBM z/VM Virtual Machine Operating System Systems Management Application Programming Interface (sometimes called VSMAPI).
  • See http://publib.boulder.ibm.com/infocenter/zvm/v5r3/topic/com.ibm.zvm.v53.dmse6/hcsl8b20.htm for more information about VSMAPI version 5.3.

Planned Development

  1. Implement the Access and Functions class library. (DONE)
    • parameter creation
    • parameter marshalling
    • communication with VSMAPI host
  2. Implement VSMAPI Functions in terms of the access class library. (IN PROGRESS | INCREMENTAL RELEASES TO 1.0)
    1. Support z/VM R5.3 (NOW)
    2. Support z/VM R5.4 (FUTURE)
    3. Support for Secure Socket Layer communications (RELEASE >= 1.1)
    4. Support user-defined custom functions (IN PROGRESS)
      • Provide Rexxx examples of custom VSMAPI extensions on the VSMAPI host itself (TODO)
      • PigGen autogenerator allows user to extend PigIron to support such custom extensions without knowing Java (DONE) (pigiron-0.2)
  3. Create Business Objects based on the Access and Functions class library (TODO)
  4. Create GUI widgets or Web objects based on Business Objects (TODO)
  5. Create GUI or Web management app based on Widgets and/or Web objects (TODO)

An important possibility is support for Secure Socket Layer communications with the VSMAPI Host.
